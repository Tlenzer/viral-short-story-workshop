# 标题与入口公式

这份参考只解决两件事：

1. 标题怎么让人点进来
2. 开头第一口怎么把人留住

开源版只保留抽象公式和匿名模板，不引用现成作品标题。

## 一、标题先解决什么

一个能跑出来的标题，至少要让读者一眼看懂：

1. 主角是谁
2. 发生了什么事
3. 哪里不对劲
4. 后面会炸成什么样

如果标题只能说明设定，不能说明冲突和后果，就还不够强。

## 二、标题高频公式

### 公式 1：关系 + 事件 + 反常点

适合情绪受害型开局。

通用写法：

`我和谁是什么关系 + 对方做了什么 + 这件事离谱在哪`

### 公式 2：异常规则 + 主角反应 + 后果预告

适合认知错位型开局。

通用写法：

`这个世界/这件事哪里不对 + 我因此发现了什么 + 后面会炸开什么`

### 公式 3：外部声音 + 现实局面 + 冲突方向

适合热帖、弹幕、系统提示、围观评论类开局。

通用写法：

`外部声音说了什么 + 现实里正在发生什么 + 这条判断会造成什么后果`

### 公式 4：身份反差 + 高压事件 + 结果尾巴

适合主角身份本身有记忆点的题材。

通用写法：

`我是谁 + 我卷进了什么高压局面 + 这件事最后会砸到谁头上`

## 三、标题自检

- 有没有具体动作，而不是空概念
- 有没有关系，而不是只有设定
- 有没有损失、危险或值钱后果
- 有没有反常点，而不是平铺直叙
- 读者能不能一句话复述冲突

## 四、开头只做四件事

1. 抛出事件或异常
2. 明确主角站位
3. 给读者一个能站队的理由
4. 立一个马上会炸开的钩子

开头不是拿来证明文笔的。
开头是拿来制造停留的。

## 五、开头模板

### 模板 A：事件先行

```text
那天，____发生了。
我原本还在忍。
直到对方当众____，我才知道这件事不可能善了。
```

### 模板 B：异常先行

```text
我最早发现不对劲，是在____的时候。
所有人都觉得____很正常，只有我知道这不对。
可真正让我发冷的，是接下来出现的____。
```

### 模板 C：外部声音先行

```text
我刷到/听到/收到一条____。
它说____。
而此刻，我面前正在发生的事，正好和这句话撞上了。
```

### 模板 D：身份反差先行

```text
我是____。
按理说，我最不该卷进____。
可偏偏在____那天，事情找上了门。
```

## 六、开头禁区

- 不要先写大段背景
- 不要先写天气、回忆和抒情
- 不要前三句都在解释设定
- 不要把真正矛盾拖到几百字后
- 不要只把标题换个说法再写一遍

## 七、最低执行要求

- 标题至少准备 5 个版本
- 开头至少准备 3 版黄金三句
- 标题和开头必须服务同一个入口
- 标题卖什么，正文第一段就接什么
